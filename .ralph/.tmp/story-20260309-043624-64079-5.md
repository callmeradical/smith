### US-005: Support skills volume mount contract
Status: in_progress
Depends on: US-001

Description:
As a loop executor, I want a stable in-container skills mount path so external skill volumes can be attached consistently.

Acceptance Criteria:
- [ ] Create and document fixed skills directory `/home/dev/.codex/skills` with runtime-user read access.
- [ ] Ensure runtime behavior does not overwrite mounted content at startup.
- [ ] Example: with `-v $(pwd)/tmp-skills:/home/dev/.codex/skills`, container lists mounted files from host.
- [ ] Negative case: starting container without mounted skills must not fail; path exists and remains empty/readable.
- [ ] Add a mount-path smoke test script that validates both mounted and unmounted scenarios.
