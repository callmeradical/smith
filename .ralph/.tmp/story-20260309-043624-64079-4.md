### US-004: Bundle internal binaries via multi-stage build
Status: in_progress
Depends on: US-001

Description:
As a platform engineer, I want internal binaries compiled in a builder stage and copied into runtime so builds are repeatable and runtime remains slim.

Acceptance Criteria:
- [ ] Implement multi-stage Docker build where internal binaries are produced in a dedicated builder stage and copied into the runtime stage.
- [ ] Place binaries in a documented runtime directory on PATH (for example `/usr/local/bin`).
- [ ] Example: runtime container executes each required internal binary with `--version` successfully.
- [ ] Negative case: if a binary is present in builder but not copied to runtime, smoke verification fails and names the missing binary.
- [ ] Document builder inputs and expected artifacts so future binary additions follow the same pattern.
