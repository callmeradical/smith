### US-004: Add CLI workflows for generate import export and validate
Status: in_progress
Depends on: US-001, US-002, US-003

Description:
As an operator, I want the `smith --prd` workflow to generate, import, export, and validate PRDs from one CLI entrypoint so PRD authoring fits existing Smith usage.

Acceptance Criteria:
- [ ] Extend the `smith` CLI so users can generate canonical PRD JSON interactively, import markdown to JSON, export JSON to markdown, and validate existing PRD artifacts.
- [ ] Keep `.agents/tasks/prd.json` as the default workspace target when an explicit output path is not provided.
- [ ] Example: `smith --prd --from-markdown docs/feature.md --out .agents/tasks/prd.json` writes canonical JSON and prints a success message referencing the saved path.
- [ ] Negative case: `smith --prd validate .agents/tasks/prd.json` exits non-zero and prints machine-readable diagnostics when the PRD is not execution-ready.
- [ ] CLI tests cover argument validation, default output paths, success flows, and failure exit codes.
