### US-001: Define Alpine-based base image skeleton and runtime user
Status: in_progress
Depends on: None

Description:
As a platform engineer, I want a canonical Alpine-based Dockerfile skeleton so every loop starts from a consistent runtime baseline.

Acceptance Criteria:
- [ ] Create or update Dockerfile to use an Alpine base image and define a non-root runtime user with a stable home directory.
- [ ] Set default working directory and shell behavior required by loop execution.
- [ ] Example: running `docker run --rm loop-base:local sh -lc 'id && pwd'` shows non-root user and expected working directory.
- [ ] Negative case: if Dockerfile uses non-Alpine base, validation fails with explicit message and build is rejected.
- [ ] Document the base image tag and rationale in repo docs.
