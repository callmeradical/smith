### US-003: Render stable markdown from canonical PRD JSON
Status: in_progress
Depends on: US-001, US-002

Description:
As a Smith user, I want canonical PRD JSON rendered back into stable markdown so the same requirements can be reviewed, edited, and shared in human-readable form.

Acceptance Criteria:
- [ ] Implement markdown export from canonical JSON using a stable section order and deterministic story formatting.
- [ ] Ensure exported markdown includes project overview, goals, non-goals, success metrics, rules, quality gates, and all stories with dependencies and acceptance criteria.
- [ ] Example: exporting a valid canonical JSON PRD yields markdown that is readable in Git and re-imports into equivalent canonical JSON for supported sections.
- [ ] Negative case: attempting export from invalid canonical JSON fails with the same validation diagnostics rather than writing misleading markdown.
- [ ] Round-trip tests verify JSON to markdown to JSON preservation for story IDs, dependency ordering, and quality gates.
