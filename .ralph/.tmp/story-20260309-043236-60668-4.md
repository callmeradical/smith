### US-004: Add interactive command controls to pod detail UI
Status: in_progress
Depends on: US-002, US-003

Description:
As an operator, I want a command input in the pod detail page so I can attach and run terminal commands directly from the console.

Acceptance Criteria:
- [ ] Route `#pod-view/:loopID` includes explicit components: attach button, detach button, command input, run button, status indicator, and runtime target summary.
- [ ] UI states are defined and implemented: `idle`, `attaching`, `attached`, `executing`, `detaching`, and `error`.
- [ ] Command submission supports Enter key and Run button, disables controls while executing, and clears input only on success.
- [ ] Example: user opens a running loop in pod detail, clicks attach, runs `pwd`, and sees command output appear in journal panel.
- [ ] Negative case: user opens non-active loop; attach and command controls remain disabled with visible reason message from runtime endpoint.
- [ ] Playwright test validates route behavior and control state transitions on success and failure paths.
