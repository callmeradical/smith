### US-006: Add reproducible verification and security gates
Status: in_progress
Depends on: US-002, US-003, US-004, US-005

Description:
As a maintainer, I want deterministic quality gate commands for build, smoke, lint, vulnerability scanning, and SBOM generation so image quality is enforceable.

Acceptance Criteria:
- [ ] Provide script or documented command set that runs all top-level `qualityGates` in order and exits non-zero on first failure.
- [ ] Include lint tooling invocation commands (`hadolint Dockerfile` and `shellcheck scripts/*.sh`) and required setup notes.
- [ ] Include security scan command `trivy image --severity CRITICAL --exit-code 1 loop-base:local` and SBOM command `syft packages loop-base:local -o spdx-json > artifacts/sbom-loop-base.spdx.json`.
- [ ] Example: running the full gate sequence after a valid build completes successfully and writes the SBOM artifact file.
- [ ] Negative case: introducing a known critical-vulnerability image tag causes trivy gate to fail with non-zero exit code.
- [ ] No CI/CD workflow files are modified in this story; only local/reusable command definitions are added.
