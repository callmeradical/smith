### US-002: Attach and detach terminal sessions against runtime target
Status: in_progress
Depends on: US-001

Description:
As an operator, I want attach/detach to represent a real session bound to the resolved runtime target so command execution is controlled and auditable.

Acceptance Criteria:
- [ ] Update `POST /v1/loops/{id}/control/attach` to require runtime resolution success and persist session metadata including runtime target reference.
- [ ] Update `POST /v1/loops/{id}/control/detach` to close only existing actor attachment and append corresponding audit/journal records.
- [ ] Example: attach from `terminal=console-pods` on a running loop returns status `attached` and increments attach count for the actor.
- [ ] Negative case: attach attempt when runtime pod is not Running returns HTTP 409 and does not create attached session state.
- [ ] Attach/detach tests assert audit metadata includes actor, terminal source, and runtime target identity.
