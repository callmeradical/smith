### US-003: Execute line commands inside active loop container
Status: in_progress
Depends on: US-002

Description:
As an attached operator, I want `control/command` to execute inside the loop container and stream results into the loop journal so I can interact from console.

Acceptance Criteria:
- [ ] Implement pod exec integration using existing Kubernetes client-go dependencies (`k8s.io/client-go/tools/remotecommand`) without introducing a new service.
- [ ] Run submitted commands via shell wrapper (for example `/bin/sh -lc`) and capture stdout/stderr plus exit status.
- [ ] Set command response `delivered=true` when exec is attempted and include exit code/result metadata.
- [ ] Example: command `echo hello` appends journal output containing `hello` and returns successful exit code.
- [ ] Negative case: actor submits command without attach and receives HTTP 409 with no pod exec call performed.
- [ ] Negative case: command over max size returns HTTP 400 with validation error and audit entry tagged as rejected.
