### US-002: Install Codex CLI in runtime image
Status: in_progress
Depends on: US-001

Description:
As a loop runtime consumer, I want Codex CLI preinstalled so loop agents can run without manual bootstrapping.

Acceptance Criteria:
- [ ] Add Codex CLI installation steps to Docker build and ensure binary is available on PATH in final stage.
- [ ] Use latest install channel during build (no version pin), as specified for v1.
- [ ] Example: `docker run --rm loop-base:local codex --version` returns a valid semantic version string.
- [ ] Negative case: if install step fails or binary is missing from PATH, smoke test exits non-zero and story is not complete.
- [ ] Record the installation command in Dockerfile comments or docs for maintainability.
