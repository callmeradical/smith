### US-006: Add execution-readiness linting for Ralph-sized stories
Status: in_progress
Depends on: US-001

Description:
As a PRD author, I want readiness linting that checks whether stories are appropriately scoped for a single Ralph iteration so generated work stays implementable.

Acceptance Criteria:
- [ ] Add semantic checks for likely oversized stories, missing negative cases, weak acceptance criteria, and incomplete dependency ordering.
- [ ] Diagnostics distinguish blocking failures from warnings and provide actionable suggestions for splitting or clarifying stories.
- [ ] Example: a story with explicit route or package changes, concrete acceptance criteria, and a negative case passes readiness linting.
- [ ] Negative case: a story that bundles CLI, API, and UI rewrites without dependencies is flagged with readiness diagnostics recommending story splits.
- [ ] Tests cover warning-only cases, blocking cases, and stable diagnostic codes for automation consumers.
