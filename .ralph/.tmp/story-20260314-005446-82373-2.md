### US-002: Normalize markdown PRDs into canonical JSON
Status: in_progress
Depends on: US-001

Description:
As a user authoring requirements in markdown, I want Smith to normalize markdown into canonical PRD JSON so autonomous workflows can consume a deterministic structure.

Acceptance Criteria:
- [ ] Implement markdown parsing that maps supported headings and task structures into the canonical PRD JSON model rather than task-only loop drafts.
- [ ] Support import of goals, non-goals, success metrics, rules, quality gates, and ordered stories when those sections are present in markdown.
- [ ] Example: a markdown PRD containing explicit story sections and quality gates produces canonical JSON with matching story IDs, ordering, and gate commands.
- [ ] Negative case: markdown missing required canonical content such as any usable stories or any quality gates produces a failed validation report with suggested fixes.
- [ ] Fixture-based tests cover well-formed markdown, partially structured markdown, and malformed markdown inputs.
