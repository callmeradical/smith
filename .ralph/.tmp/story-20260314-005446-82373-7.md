### US-007: Document and verify the PRD authoring workflow end to end
Status: in_progress
Depends on: US-003, US-004, US-005, US-006

Description:
As a maintainer of systems using Smith for autonomous development, I want clear docs and end-to-end verification for PRD authoring and validation so teams can adopt the workflow consistently.

Acceptance Criteria:
- [ ] Document canonical JSON expectations, markdown authoring rules, validation semantics, and CLI examples in repository docs.
- [ ] Add an end-to-end fixture or script that exercises markdown import, JSON validation, markdown export, and PRD ingress using shared validation rules.
- [ ] Example: documented workflow shows how a markdown PRD becomes `.agents/tasks/prd.json`, validates cleanly, and becomes eligible for autonomous execution.
- [ ] Negative case: docs and fixtures include an invalid PRD example showing the exact diagnostic behavior and blocked execution path.
- [ ] Verification covers the documented workflow under `go test ./...`, `./scripts/validate-acceptance.sh`, and `make ci-local-act`.
